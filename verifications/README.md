# Verifications

MVPをまたいで再利用する技術検証、実装メモ、最小実装を蓄積します。

ここにあるものは、各MVPから直接依存しない参照用の資産です。MVPへ使う場合は、必要なコードと知見をMVP内へコピーし、MVP側の `docs/knowledge/` または `experiments/` に採用メモを残します。

## 置くもの

- 外部ログインやOAuth連携
- Webhook署名検証
- 決済サンドボックス接続
- メール送信、SMS送信、通知
- ストレージやアップロード
- Codexが繰り返し躓きやすい環境構築や最小実装

## 標準構成

```text
verifications/
└── <topic>/
    ├── README.md
    ├── findings.md
    └── minimal-impls/
```

## 運用

- 公式ドキュメントのURLと確認日を残します。
- 実行できる最小実装には、起動方法と必要な環境変数を残します。
- 秘密情報や実データは入れません。
- 古くなりやすい外部サービス仕様は、実装前に公式ドキュメントを再確認します。
