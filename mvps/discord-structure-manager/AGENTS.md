# AGENTS.md

このファイルは、このMVPディレクトリ配下に適用されます。

## 作業前に確認するもの

- `README.md`
- `docs/brief.md`
- `docs/runbook.md`
- 既存の起動・テスト・デプロイ手順

## MVP内ルール

- 変更は原則としてこのMVPディレクトリ内に閉じます。
- 他MVPやトップレベルに影響する変更は、必要性を確認してから行います。
- このディレクトリの中身は、将来的に別リポジトリのルートへ移す前提で保ちます。
- 親ディレクトリのファイルを実行時・ビルド時の必須依存にしません。
- 仕様や重要な判断が変わった場合は、`README.md` または `docs/decisions/` に記録します。
- 実装中に得た知見、失敗した試行、採用理由は `docs/knowledge/` に残します。
- MVP固有の技術検証、スパイク、最小実装は `experiments/` に置きます。
- 複数MVPで再利用できる知見は、トップレベルの `verifications/` にも還元します。
- 環境変数は `.env.example` に名前と用途だけを書き、秘密情報は入れません。
- サンプルデータは `data/samples/` に置き、実データや個人情報はコミットしません。
- 検証に使ったコマンドは作業結果に明記します。
