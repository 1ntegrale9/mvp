# MVP Brief

## Problem

Discordサーバーの権限設計は、チャンネル、カテゴリ、ロール、権限上書きが分散しており、まとめて整理しづらい。特に複数チャンネル/複数ロールへ同じプリセットを反映する作業が手間になりやすい。

## Target User

Discordコミュニティ運営者、サーバー管理者、Bot開発者。

## Hypothesis

常時編集可能な下書き画面、まとめて適用、プリセット保存、複数の並べ替えUIを持つローカルWebアプリがあれば、実際のDiscord API連携前でも権限設計の作業感を検証できる。

## Scope

### In

- チャンネル/カテゴリの並べ替え
- チャンネルのカテゴリ同期状態の表示
- チャンネルごとのカテゴリ同期ON/OFF
- フォーラムチャンネルの表示
- テキスト/フォーラムチャンネル配下のスレッド表示
- スレッドの画面上ボタンによる遅延取得
- ロールの並べ替え
- ドラッグアンドドロップ
- 上下移動ボタン
- 移動先セレクトメニュー
- 権限プリセットの保存/適用
- 複数チャンネル/複数ロールへの一括適用
- 編集、保存、破棄のUX
- Discord APIのread-only構成取得
- サーバー切り替え

### Out

- Discord APIへの実保存
- Bot OAuth認証
- 監査ログ連携

## Success Criteria

ローカルで起動し、主要操作をブラウザで実行できる。

## Validation Plan

ブラウザでチャンネル移動、ロール移動、プリセット一括適用、カスタムプリセット保存、保存/破棄を確認する。
