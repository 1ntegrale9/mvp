# Discord-like Sidebar Layout

## Context

チャンネル管理、ロール管理、権限プリセット管理を別ページに分けた後、各ページの中で一覧と詳細の距離が近いほうが操作しやすいことが分かった。実サーバーのチャンネル数やロール数が多い場合、対象を選ぶ場所と編集する場所を明確に分ける必要がある。

## Decision

- チャンネル管理、ロール管理、権限プリセット管理の別ページ仕様は維持する。
- 各ページの左ドロワーに、そのページで扱う一覧だけを表示する。
- 一覧行のボタンは、チェック、展開、ドラッグハンドル、名前選択に絞る。
- 上下移動、カテゴリ同期、スレッド取得、移動先セレクト、プリセット反映は中央〜右の作業面に置く。
- PCとスマートフォンで別UIを作らず、同じ一覧ドロワーを開閉できるようにする。
- 一覧の開閉は、開いた状態でも閉じた状態でも同じ縦長レールボタンで行う。ボタンは常に左端に固定し、開いた状態では一覧側にレール分の余白を持たせて内容を隠さない。
- 管理ページ切替、保存、破棄、初期化はヘッダー内でアイコン表示にする。ページ名はサーバー文脈側へ集約し、作業面側には選択状態だけを表示する。モバイルでは補足文を非表示にして、ページ名と状態だけを残す。
- ヘッダー左上はサーバー文脈を優先し、サーバーアイコン、サーバー名、オーナー名、現在の管理ページ、管理対象切替、サーバー切替をまとめて置く。オーナー名は現状のread-only取得データに含まれないため未取得表示にする。

## Notes

Discord風にドロワー側は濃い背景の密な一覧、右側は明るい設定面にした。視覚的な雰囲気は参考にするが、既存MVPの保存/破棄、read-only取得、権限プリセット操作はそのまま保つ。
