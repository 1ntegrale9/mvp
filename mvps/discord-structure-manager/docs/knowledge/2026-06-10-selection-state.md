# Selection State Notes

## Context

権限管理UIでは、チェック済みのチャンネルとロールが「一括反映の対象」を意味する。初期表示でどれかにチェックが入っていると、サンプル表示なのかユーザー選択なのかが曖昧になる。

## Decision

- 初期状態と初期化後は、チャンネルもロールも0件選択にする。
- 選択は0件まで外せるようにする。
- 権限変更、プリセット保存、プリセット反映、カテゴリ同期の一括操作は、必要な対象が選択されるまで無効化する。
- アクティブ行の強調も初期状態では出さない。

## Result

未選択状態を明示し、ユーザーがチェックを入れたものだけを一括反映対象として扱えるようにした。

## Category Selection

カテゴリ行のチェックボックスは、カテゴリ自体ではなく、そのカテゴリ内の子チャンネルをまとめて選択/解除する用途にする。カテゴリ行には `選択数/総数` を表示し、部分選択でも現在の対象数が分かるようにする。

## Category Collapse

カテゴリ行には、選択用チェックボックスとは別に開閉ボタンを置く。閉じたカテゴリの子チャンネルはリストから隠すが、カテゴリ内の一括選択は閉じた状態でも使えるようにする。閉じたカテゴリ内のチャンネルがアクティブだった場合は、見えない行を編集している状態を避けるためアクティブ状態を解除する。

## Channel Type Badges

チャンネル種別バッジは、テキスト `T`、フォーラム `F`、アナウンス `A`、ステージ `S`、スレッド `Th` として表示する。カテゴリ `C` とボイス `V` は既存の短縮表示を維持する。Discord API由来の `guild_forum`、`public_thread` などの種別名も同じ表記へ寄せる。

チャンネル名の下には小さくIDを表示する。Discord API連携後は表示名が重複する可能性があるため、選択・権限反映対象を確認しやすくする。DiscordのチャンネルIDとロールIDは18〜20桁の数字として扱い、サンプルデータでもslugではなく数字IDを内部キーにする。

## Thread Display

テキストチャンネルとフォーラムチャンネルの配下にスレッドを表示する。`parentId` はカテゴリだけでなく親チャンネルも指せるため、カテゴリ同期の対象は「親がカテゴリのチャンネル」だけに限定する。スレッド行にはカテゴリ同期ボタンを出さず、権限編集時は親チャンネルの有効権限を参照・反映する。

カテゴリを閉じた場合は、そのカテゴリ配下の通常チャンネルだけでなく、さらに下のスレッドも隠す。テキスト/フォーラムチャンネルを閉じた場合は、その配下のスレッドだけを隠す。カテゴリ内一括選択は、スレッドも含めてカテゴリ配下のすべての非カテゴリ行を対象にする。
