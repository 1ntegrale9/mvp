# Always Editable Draft

## Context

このMVPはDiscordへ書き込みを行わず、画面内の変更はローカル下書きとして扱う。編集ON/OFFを分けると、各ページで最初に編集ボタンを押す必要があり、チャンネル管理、ロール管理、権限プリセット管理をまたぐ作業の流れが重くなる。

## Decision

- 編集状態のON/OFFボタンは置かない。
- チャンネル移動、ロール移動、カテゴリ同期、詳細入力、権限編集、プリセット保存/適用は常に操作可能にする。
- Save / Discard は残し、未保存の変更を下書きとして管理する。

## Notes

Discord APIへのwriteはまだ行わないため、常時編集可能でも外部副作用は発生しない。今後write操作を入れる場合は、保存前確認や差分レビューを別途設計する。
