## 検証したい仮説

Discordのチャンネル構成、ロール構成、権限プリセットをGUIで編集し、まとめて反映できると、サーバー初期設計や権限整理の作業が速くなる。

## 対象ユーザー

Discordサーバー管理者、コミュニティ運営者、Bot/運営ツール開発者。

## 成功条件

- 常時編集可能な下書き画面で、構成変更と権限変更を行える
- 保存/破棄の差分UXが分かる
- チャンネル管理、ロール管理、権限プリセット管理の各ページで、左ドロワーの一覧から対象を選び、中央〜右の作業面で詳細と設定を編集できる
- ドラッグアンドドロップ、上下移動、移動先セレクトメニューで並べ替えできる
- チャンネルのカテゴリ同期状態を表示し、同期ON/OFFを切り替えられる
- フォーラムチャンネルと、テキスト/フォーラムチャンネル配下のスレッドを表示できる
- スレッドは画面上の取得ボタンから必要なチャンネルだけ安全に読み込める
- 権限プリセットを複数チャンネル/ロールへまとめて反映できる
- カスタムプリセットを保存して再利用できる

## 技術スタック

Static HTML / CSS / JavaScript。Discord読込データやスレッド取得を使うローカル検証では、GET専用のNodeサーバーで配信する。

## ローカル起動

```sh
node scripts/serve-readonly.mjs
```

ブラウザで `http://localhost:5173` を開く。

静的UIだけなら `apps/web` で `python3 -m http.server 5173` でも確認できます。ただし画面上のスレッド取得ボタンはNodeサーバー起動時だけ動作します。
